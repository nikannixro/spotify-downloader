# Fix timeout: change Session.invoke default timeout

## Why monkey-patching fails
`invoke.py:80` calls `session.invoke(query, retries, timeout, sleep_threshold)` — all positional.
`save_file.py:109` calls `session.invoke(data)` — uses default timeout.
Our patch adds `timeout=` keyword → conflicts with positional calls.

## Fix
Replace the broken monkey-patch in both `main.py` with a one-liner that changes the default timeout:

```python
    # Increase default timeout for file uploads (default 15s too low)
    import pyrogram.session.session as _sess
    _sess.Session.invoke.__defaults__ = (10, 20, 10)  # retries, timeout, sleep_threshold
```

This changes default timeout from 15→20. Callers that pass timeout explicitly (invoke.py) keep their value. Callers that use default (save_file.py worker) get 20s.
